HTML整理

发表于 2019-03-19 | 分类于 HTML整理

1. 块级元素和内联元素

  • 块级元素：在页面中以块的形式展现；相对于前面的内容它会出现在新的一行，其后的内容也会被挤到下一行展现
  • 内联元素：通常出现在块级元素中并包裹文档内容的一小部分，不会导致文本换行，比如<em>斜体、<a>超链接和<strong>

2. 实体引用

在HTML中，字符<,>,",'和&是特殊字符，本身是HTML语法的一部分，想在文本中使用时必须用字符引用；常用的如下：

原义字符 等价字符引用
< &lt;
> &gt;
“ &quot;
‘ &apos
& &amp

3.CSS和JavaScript的引入

  • <link>元素经常位于文档的头部，引入css文件，该元素有2个属性，rel="stylesheet"表明这是文档的样式表，而href包含了样式表文件的路径
  • <script>部分放在文档尾部是一个更好的选择，确保在加载脚本之前浏览器已经解析了HTML内容

4.尽可能使用相对链接

  • 当链接到同一网站的其他位置时，应该使用相对链接
  • 当链接到另一个网时，需要使用绝对链接

理由如下：

  1. 检查代码容易得多：相对URL通常比绝对URL短得多，阅读容易
  2. 链接到同一网站的其他位置：当使用绝对URL时，浏览器首先通过DNS查找服务器得真实位置，然后再转到该服务器并查找所请求的文件；如果使用相对URL，浏览器只在同一服务器上查找被请求的文件，可以避免不断地让浏览器做额外的工作，那样子意味着效率会降低

    5.不常见的HTML元素

  • 描述列表使用<dl>(description list)标签，每一项都用<dt>(description term)，每个描述都用<dd>(description description)，一个<dt>可以同时有多个描述<dd>

6.HTML布局元素细节

  • <header>：标题栏，一般是网站的全局页眉
  • <nav>：导航栏
  • <main>：主内容，可以有各种子内容区段<article>、<section>和<div>，用于存放每个页面独有的内容，每个页面上只能用一次，且直位于<body>
  • <article>：包含的内容即一篇文章
  • <section>：与<article>类似，但它更适用于组织页面使其按功能分块
  • <aside>：侧边栏，经常嵌套在<main>中
  • <footer>：页脚

    7.无语义元素

  • <span>：内联无语义元素
  • <div>：块级无语义元素

    8.换行与水平分割线

  • <br>：换行
  • <hr>：水平分割线

    9.图片

  • <img>和<video>这样的元素也叫替换元素，因为元素的内容和尺寸由外部资源所定义，而不是元素自身，可通过宽度和高度属性来指定图片的高度和宽度
  • 备选文本：属性alt，值为对图片的文字描述，用于在图片无法显示或不能被看到的情况
  • 说明文字：使用HTML5中的<figure>和<figcaption>元素，为图片提供一个语义容器，在标题和图片之间建立清晰的关联
  • <figure>里不一定要是一张图片，只要是个独立内容单元，可以是几张图片、一段代码、音视频等

    10.web中的音频视频

  • 标签之间一般设有备选内容，如果浏览器不支持该标签将会显示备选内容
  • HTML5新特性：<video>和<audio>
  • <video>：常使用<source>获取资源
属性 描述
src 指向想要嵌入网页当中的视频资源
controls 用户能够控制视频和音频的回放功能
width和height 控制视频的尺寸
autoplay 使视频和音频内容立即播放，即使页面的其他部分还没有加载完全，建议不要应用
loop 视频或音频文件循环播放
muted 媒体播放时默认关闭声音
preload 用来缓冲较大的文件，有none、auto和metadata
  • <audio>：与video很像，但是不支持width/height属性

11.<iframe>

  • 用于嵌入其他网页(不熟悉，日后学习补充)
