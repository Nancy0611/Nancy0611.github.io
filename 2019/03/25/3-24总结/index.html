发表于 2019-03-25 | 分类于 总结

关于判断类型

  • typeof只能判断基本类型，即number、string、underfined、boolean；无法区别引用类型，对于数组、对象、函数返回值只有object/function
  • Object.prototype.toString判断原生引用类型，分为null,string,boolean,number,underfined,array,function,object,date,math；无法判断自定义类的实例，只能通过instanceof

判断是否为数组的方法

1. Array.isArray()
  • 在检查数组类型时用isArray比instanceof好，因为前者支持跨越iframes

如果该方法本身不可用，则手动创建

2. instanceof
  • instanceof用于判断一个变量是否是某个对象的实例；运算符的第一个变量是一个对象A，第二个变量一般是一个函数B。
  • 判断原则是：沿着A的proto这条线来找，同时沿着B的prototypr这条线来找，如果两条线能找到同一个引用，即同一个对象，那么就返回true。如果找到终点还未重合，则返回false。
3. construction
  • construction属性返回对创建此对象的数组函数的引用
4. Object.prototypr.toString.call(arr)
注意
  • instanceof和constructor被判断的array必须是在当前页面声明的！比如：一个父页面有一个框架，框架中引用了一个子页面，在子页面声明了一个array，并将其赋值给父页面的一个变量，这时候判断该变量Array==object.constructor返回false
  • 原因：array为引用型数据，在传递过程中，仅仅是引用地址的传递；每个页面的Array的原生对象所引用的地址是不一样的

深浅拷贝的区别及实现

转发该博文 听风是风

  • 所谓深拷贝本身是针对较为复杂的object类型数据
  • 拷贝的是对象各个层级的属性
  • concat方法和slice都只是拷贝了一级属性，拷贝不彻底，都不是真正的深拷贝
    1. 递归复制所有层级属性
2. JSON对象的parse和stringify
3. JQuery的extend方法
  • $.extend([deep],target,object1[,objectN])
  • deep表示是否深浅拷贝，true为深拷贝，false为浅拷贝
  • target为Object类型，目标对象，其他对象的成员属性将被附加到该对象上
  • object为第一个以及第N个被合并的对象
