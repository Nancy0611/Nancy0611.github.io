webpack入坑笔记（1）

一、webpack安装、创建项目

  1. 从node.js官网下载最新安装包进行安装

    • npm随nodejs安装包一起安装的包管理工具
    • 允许用户从npm服务器下载/上传编写的包或命令行程序
    • 命令行工具输入npm -v检测是否安装

  2. 使用npm安装webpack

  3. 进入项目目录，输入命令行npm init生成package.json文件

  4. 输入命令npm install webpack --save-dev为项目添加webpack依赖

  5. 创建一个静态文件hello.js，里面包含一个hello函数和通过require引入world.js

二、loader的使用

webpack本身只能处理JavaScript模块，如果要处理其他类型的文件，就需要使用loader进行转换，Loader可以理解为是模块和资源的转换器，它本身是一个函数，接受源文件作为参数，返回转换的结果。这样我们就可以通过require来加载任何类型的模板或文件，比如JSX、LESS或图片

  1. 在hello.js中通过require(‘./style.css’)引入style.css，执行npx webpack hello.js --output-filename hello.bundle.js --output-path . --mode development，报错如下：

    报错提示需要一个loader转换器来处理css样式

  2. 安装css的loader转换器和style的loader转换器

    安装完成后输入编译hello.js命令，运行后仍然发现报错

  3. webpack只能处理javascript模块，处理css需要借助于loader转换器，在require()引用模块的时候添加loader转换require('css-loader!./style.css');

    说明：style-loade和css-loader是工具名称

    ！感叹号是分隔符，表示两个工具都参与处理

    ？问好跟url的问号一样，就是后面要跟参数的意思

    编译打包成功

    查看hello.bundle.js文件，多出了css样式这一块

  4. 在项目目录下新建一个index.html，并且引入hello.js打包后的hello.bundle.js

  5. 修改hello.js，运行hello('hello webpack')函数，在编译一次hello.js到hello.bundle.js

  6. 运行index.html，页面弹出”hello webpack!”，但是页面的背景色并没有改变，也就是说css样式没有生效，还需要再require引入css的时候使用style-loader，编辑完后重新编译打包

  7. 刷新index页面，页面变成了pink的背景色

    style.css中的样式被使用style标签插入到了head里面，这是由style-loader实现的

    css-loader是允许webpack识别.css的文件

    style-loader是将webpack识别完的css文件中的内容，在编译完运行文件的时候，将这些css用style标签包起来嵌在head内

三、webpack.config.js配置文件

webpack在执行的时候，除了在命令行输入参数，还可以通过指定的配置文件来执行。默认情况下输入webpack，会搜索当前目录的webpack.config.js文件，如果配置文件为其他，需通过--config选项来指定配置文件

  1. webpack.config.js为webpack的配置文件

    输入命令webpack即可按照webpack.config.js中的配置项进行编译

  2. 如果将webpack.config.js重命名为webpack.dev.config.js，则直接执行默认的webpack命令会找不到相应的配置文件，需借助于webpack的--config选项来指定配置文件

    命令：webpack --config webpack.dev.config.js

  3. 如果要查看编译进度、打包的模块之类的，可以在packsge.json中的scripts标签内通过webpack的属性值来指定

    输入命令npm run webpack来编译打包，通过npm run执行对应名称的脚本，脚本内容执行时，webpack命令会执行指定配置文件
