清除浮动

发表于 2019-04-18 | 分类于 CSS整理

一、相关属性

1. clear
  • clear属性只会对自身起作用，而不会影响其他元素。如果一个元素的右侧有一浮动对象，而这个元素设置了不允许右边有浮动对象，即clear：right，则这个元素会自动下移一格，达到本元素右边没有浮动对象的目的
值 属性
none 默认值，允许两边都可以有浮动
left 不允许左边有浮动对象
right 不允许右边有浮动对象
both 不允许有浮动对象
2. overflow
  • 当父元素的高度是靠子元素撑开的时候，子元素浮动时，则在父元素使用overflow:hidden可以清除浮动，使得父元素的高度依旧是靠子元素撑开的
  • 当父元素自身设置了height属性值，则在父元素使用overflow:hidden可以使子元素超出父元素的那部分隐藏
  • 注：动手试了下后发现，hidden所谓的隐藏溢出，是指内容溢出边框外的部分被隐藏，边框之内的还是可见的（包括内容区和内边距）

二、伪元素

  • 伪元素：允许我们添加额外元素而不扰乱文档本身
  • :before和:after发布于CSS2.1，最初语法是使用:(一个冒号)
  • 在CSS3中修订后的伪元素使用::(两个冒号)以区分伪元素和伪类
  • 添加的内容不会出现在DOM中，不会改变文档内容，不可复制，仅仅是在css渲染层加入，所以不要用它们来展示有实际意义的内容，尽量使用它们显示修饰性内容，比如图标
  • :before会在内容之前“添加”一个元素，:after会在内容之后“添加”一个元素
  • :before和::after下必须配合content属性来使用，用于在定义插入的内容，必须有值，至少是空
  • 默认情况下伪类元素的display是inline

三、浮动清除

清除浮动主要是为了解决：父元素因为子级元素浮动引起的内部高度为0的问题

  1. 父级添加overflow属性
    • 做法：父元素添加overflow:hidden
    • 原理：通过触发BFC方式，实现清除浮动
    • 缺点：内容增多的时候容易造成不会自动换行导致内容被隐藏掉，无法显示要溢出的元素，不推荐使用
  2. 额外标签法
    • 做法：在最后一个浮动标签后，新加一个标签，设置其clear:both;
    • 原理：清除了浮动，父元素自动检测子盒子最高的高度，然后与其同高
    • 缺点：添加无意义标签，语义化差，不建议使用
