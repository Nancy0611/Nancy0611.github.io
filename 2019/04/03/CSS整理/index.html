CSS整理(1)

发表于 2019-04-03 | 分类于 CSS整理

最近一周很少写笔记，过于浮躁，有些急于求成

同一个HTML元素定义了多个样式

优先权高–>低

  1. 内联样式（HTML元素内部）
  2. 内部样式表（位于head标签内部）
  3. 外部样式表
  4. 浏览器缺省设置

选择器

  • ID选择器：ID属性只能在每个HTML文档中出现一次
  • class选择器：用于描述一组元素的样式，可以在多个元素中使用

多重样式优先级理解

  • 优先级就是分配给指定的CSS声明的一个权重，由匹配的选择器中的每一种选择器类型的数值决定
  • 优先级顺序，逐级增加：
  1. 通用选择器
  2. 元素选择器
  3. 类选择器
  4. 属性选择器
  5. 伪类
  6. ID选择器
  7. 内联样式

Backgrounds

  • background-color不能继承，默认值是transparent；如果一个元素没有指定背景色，那么背景就是透明的，这样其父元素的背景才可见
  • background-image默认情况下，背景图片进行平铺重复显示，以覆盖整个元素实体；而且默认会在水平或垂直方向平铺
  • background-repeat不平铺则设为no-repeat
  • background-position属性改变图像在背景中的位置
    • 关键字：top、bottom、left、right、center；保证不超过2个关键字，一个对应水平一个对应垂直；若只有一个则另一个为center；如top left使图像放置在元素内边距的左上角
    • 百分比：图像中心与其元素中心对齐
    • 长度值：元素内边距区左上角的偏移，偏移点是图像的左上角

px-em-rem区别

  • px：像素是相对于显示器屏幕分辨率而言的
  • em：会继承父级元素的字体大小，任意浏览器的默认字体高都是16px

第一个p标签继承了父元素body的font-size默认是16px，则它的1em也就是16px；第二个p标签的父级div设置font-size为50px，在这里的p标签1em就等于50px；

  • rem：CSS3新增加的单位相对的只是HTML根元素，默认也是16px，通过它既可以做到只修改根元素就成比例地调整所有字体大小，又可以避免字体大小逐层复合的连锁反应

链接link状态（伪类）

  • a:hover必须跟在a:link和a:visited后面
  • a:active必须跟在a:hover后面
  • 顺序规则的原因
  • a:link正常，未访问过的链接
  • a:visited用户已访问过的链接
  • a:hover当用户鼠标放在链接上时
  • a:active链接被点击的那一刻

盒子模型

  • Margin（外边距）清除边框区域。Margin没有背景颜色，它是完全透明
  • Border（边框）边框周围的填充和内容。边框是受到盒子的背景颜色影响
  • Padding（内边距）清除内容周围的区域。会受到框中填充的背景颜色影响
  • Content（内容）盒子的内容，显示文本和图像
  • 盒子实际所占的宽度由“内容+内边距+边框+外边距”组成
  • CSS中可通过width和height控制内容所占矩形的大小

padding

  • padding属性可以使用长度值或百分比值，但与margin不同，不允许使用负值
  • 百分比数值是相对于其父元素的width计算的
  • 上下内边距与左右内边距一致，即上下内边距的百分数会相对于父元素宽度设置，而不是相对于高度

display与visibility

  • 隐藏元素-display:none或visibility:hidden
  • visibility:hidden隐藏的元素仍需占用与未隐藏之前一样的空间；虽被隐藏还是会影响布局
  • display:none隐藏的元素不会占用任何空间；该元素原本占用的空间会从页面布局中消失
  • display:block更改内联元素为块元素，不允许有它内部的嵌套块元素

position定位

  • 元素可以使用的顶部，底部，左侧和右侧属性定位，在没设定position属性前是无法工作的
  • static，默认值，没有定位，元素出现在正常的流中，静态定位的元素不会受到top,bottom,left,right影响
  • fixed，元素位置相对于浏览器窗口是固定位置；fixed定位使元素与文档流无关，因此不占据空间
  • relative，元素定位是相对其正常位置；原本所占的空间不会改变，相对定位元素经常被用来作绝对定位元素的容器块
  • absolute，绝对定位的元素的位置相对于最近的已定位父元素，如果元素没有已定位的父元素，那它的位置相对于;定位位置与文档流无关，所以不占据空间

居中块级元素

  • 明确设置width，左右外边距都设为auto

组合选择符

CSS3四种组合方式：

  • 后代选取器（以空格分隔）：匹配所有指定元素的后代元素
  • 子元素选择器（以大于号分隔）：只能选择某元素的直接子元素
  • 相邻兄弟选择器（以加号分隔）：需要选择紧接在另一个元素后的元素，而且二者有相同的父元素
  • 普通兄弟选择器（以波浪号分隔）

CSS布局-标准流布局

关于脱离文档流分析可看看这篇

  • 传统的HTML文档中的文本布局，从左向右从上向下显示
  • 标准流中，块级元素独占一行，垂直放置，行级元素在水平方向上一个接一个的排列
垂直外边距的塌陷合并问题
  1. margin合并问题：在标准流中上下两个盒子的外边距会和，即上盒子的margin-bottom会与下盒子的margin-top合并为两者之间的最大值
  1. margin塌陷问题：一个盒子包裹另一个盒子，当包裹盒子没有设置border和padding时，里边盒子的设置的上边距不会起作用

    具体表现是当两个元素嵌套到一起时，外层盒模型的margin-top取两个元素中margin-top较大的值，因为在正常的情况下内层元素是相对于外层元素进行移动，但是这时内层元素却相对于整个文档进行移动

  • margin塌陷问题解决方法
    1. 给外层元素人为加一个“棚顶”border；此时的margin就是相对于父元素的（虽然能够解决问题，但是在日常开发中不使用它，因为它在外观上对元素进行了改变）
    2. bfc–block format context块级格式化上下文；我们可以给外层元素加一个特定属性来触发bfc，触发bfc方式有以下四种：
    • overflow:hidden
    • display:inline-block
    • position:absolute
    • float:left

      一般margin合并不解决，margin塌陷要解决

文档流及定位

  • block元素：独占一行，多个block元素会新起一行
    • 可以设置width height margin padding属性
    • 默认block元素宽度自动填满其父元素宽度
    • h1,h2,h3,h4,h5,h6,p,div,dl,dt,hr,ol,ul,li,form,pre,table,td,th；
  • inline元素：不会独占一行，多个相邻的行内元素会排列在同一行内，直到一行排列不下，才会新换一行
    • 其宽度随元素的内容而改变
    • 设置width height属性无效
    • 水平方向的padding margin有效
    • 竖直方向的padding margin无效
    • em,strong,span,button,input,label,code,select,img,textarea
  • 文档流：元素排版布局过程中，元素会自动从左往右，从上往下的流式排列
  • 脱离文档流：就是将元素从普通的布局排版中拿走，其他盒子在定位的时候，会当做脱离文档流的元素不存在而进行定位
    • float:使用float脱离文档流时，其他盒子会无视这个元素，但其他盒子内的文本依然会为这个元素让出位置，环绕在周围部分无视
    • absolute：使用absolute position脱离文档流的元素，其他盒子与其他盒子内的文本都会无视它，完全无视
    • fixed
浮动元素后跟block元素&inline元素对布局的影响
  • 浮动的框之后的block元素元素会认为这个框不存在，但其中的文本依然会为这个元素让出位置。
  • 浮动的框之后的inline元素，会为这个框空出位置，然后按顺序排列
元素浮动造成的两个div覆盖或相互重叠如何解决
  • 左右结构div盒子重叠现象
    • 要么都不使用浮动
    • 要么都使用float浮动
    • 要么对没有使用float浮动的div设置margin样式
  • 上下结构div盒子重叠现象
    • 大盒子里的子元素使用了浮动，脱离了文档流，导致大盒子没被撑开，大盒子下的盒子认为大盒子没有高度（未被撑开）因此就跑上去了
    • 可给大盒子设置固定高度，但是一般不可行，因为文字内容不确定多少就不能设置固定高度
    • 使用clear清除浮动，在大盒子闭合前加个空div块并使用clear:both来表示两侧都不允许有浮动元素，这样新的空div块会下移，达到撑开父元素的目的
- 给父元素设置`overflow:hidden`来清除浮动
clear属性
值 属性
none 默认值，允许两边都可以有浮动
left 不允许左边有浮动对象
right 不允许右边有浮动对象
both 不允许有浮动对象
  • clear属性只会对自身起作用，而不会影响其他元素。如果一个元素的右侧有一浮动对象，而这个元素设置了不允许右边有浮动对象，即clear：right，则这个元素会自动下移一格，达到本元素右边没有浮动对象的目的。
