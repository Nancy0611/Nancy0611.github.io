事件

发表于 2019-04-07 | 分类于 Javascript基础

1. 事件流

事件流描述的是从页面中接收事件的顺序

1.1 事件冒泡流

  • 事件开始时由最具体的元素（文档中嵌套层次最深的那个节点）接收，然后逐级向上传播到较为不具体的节点（文档）

1.2 事件捕获流

  • 不太具体的节点应该更早接收到事件，而最具体的节点应该最后接收到事件

1.3 DOM事件流

  • DOM2级事件规定的事件流包括三个阶段
    • 事件捕获阶段
    • 处于目标阶段
    • 事件冒泡阶段

2. 事件处理程序

  • 事件就是用户或者浏览器自身执行的某种动作，比如click、load、mouseover，都是事件的名称
  • 响应某个事件的函数就叫做事件处理程序

2.1 HTML事件处理程序

  • 某个元素支持的每种事件，都可以使用一个与相应事件处理程序同名的HTML特性来指定
  • 缺点
    1. 存在时差问题：上面例子中假设在页面解析showMessage()函数之前就单击了按钮，就会引发错误
    2. 扩展事件处理程序的作用域链在不同浏览器中会导致不同结果
    3. HTML与Javascript代码紧密耦合

2.2 DOM0级事件处理程序

  • 将一个函数赋值给一个事件处理程序属性

  • 以这种方式添加的事件处理程序会在事件流的冒泡阶段被处理

  • 优点

    • 简单
    • 具有跨浏览器的优势
  • 缺点
    • 只能给一个事件绑定一个响应函数，重复绑定会覆盖之前的绑定

2.3 DOM2级事件处理程序

  • DOM2级事件定义了两个方法
    • 处理指定事件处理程序：addEventListener()
    • 删除事件处理程序：removeEventListener()
    • 接受3个参数，分别是：
      • 要处理的事件名
      • 作为事件处理程序的函数
      • 布尔值
        • true：表示在捕获阶段调用事件处理程序
        • false：表示在冒泡阶段调用事件处理程序
  • 好处：可以添加多个事件处理程序，事件处理程序会按照添加顺序触发
  • 大多数情况下，都是将事件处理程序添加到事件流的冒泡阶段

2.4 执行顺序

参考链接

  • 目标事件在哪一层，事件流就在哪一层回流，若点击的是外层outer，即使在outer事件下还有许多子孙节点，事件流都不会在oute之后往内流，此时inner上的事件不会被触发
  • 若事件是在“处于目标阶段”，执行顺序取决的不再是捕获还是冒泡，而是谁在代码中先注册
  • 如果DOM 0级和DOM 2级事件同时存在
    • 当绑定DOM 0级的事件元素不是目标元素时，DOM 0级事件的处理是在冒泡阶段处理的，并按事件注册的先后顺序执行（W3C先注册的先执行）
    • 当绑定DOM 0级事件的元素是目标元素时，则不论是捕获阶段绑定的处理函数，还是冒泡阶段绑定的处理函数以及DOM 0级事件处理函数，它们的执行顺序都是按照注册的顺序执行（W3C先注册的先执行）

3. 内存和性能

添加到页面上的事件处理程序数量将直接关系到页面的整体运行性能

  • 每个函数都是对象，都会占用内存，内存中的对象越多，性能就越差
  • 必须实现指定所有事件处理程序而导致的DOM访问次数，会延迟整个页面的交互就绪时间
  • 解决方法
    • 事件委托，减少建立连接的数量
    • 移除事件处理程序，减少内存中那些过时不用的“空事件处理程序”

3.2 移除事件处理程序

  1. 从文档中移除带有事件处理程序的元素时，原来添加到元素中的事件处理程序极有可能无法被当作垃圾回收，可以的话最好手工移除事件处理程序
  1. 卸载页面的时候，如果在页面被卸载之前没有清理干净事件处理程序，那么它们就会滞留在内存中。一般最好的做法是在页面卸载之前先通过onunload事件处理程序移除所有事件处理程序（相对于浏览器页面间来回切换或者单击刷新按钮）

4. 事件对象

在触发DOM上的某个事件时，会产生一个事件对象event，对象中包含着所有与事件有关的信息，包括导致事件的元素、事件的类型等

4.1 DOM中的事件对象

兼容DOM的浏览器会将一个event对象传入到事件处理程序中

  • event对象包含与创建它的特定事件有关的属性和方法
属性 类型 说明
currentTarget Element 其事件处理程序当前正在处理事件的那个元素
target Element 事件的实际目标
eventPhase Integer 调用事件处理程序的阶段123
preventDefault() Function 取消事件的默认行为
stopPropagation() Function 取消事件的进一步捕获或冒泡
type String 被触发的事件的类型
  • 在事件处理程序的内部，对象this始终等于currentTarget的值（事件注册到的元素上），而target则包含事件的实际目标
